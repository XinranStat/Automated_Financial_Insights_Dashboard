# MarketSenseAI: Automated Financial Insights Dashboard

MarketSenseAI is an end-to-end data pipeline that scrapes global financial news, performs AI-driven sentiment analysis, and correlates findings with market data. It transforms raw reporting from Financial Times and The Economist into a strategic executive dashboard.

---

## Core Features

* Automated ETL: Scrapes RSS feeds with custom time-filtering (48 hours for FT, 90 days for The Economist).


* AI Classification: Tags articles into Macro, Tech/AI, Geopolitics, or Corporate Finance using Gemini 2.5 Flash.


* Strategic Scoring: Assigns sentiment scores (-1.0 to +1.0) based on operational and strategic impact.


* Market Correlation: Maps news events against Nasdaq 100 and 10Y Treasury yield volatility.


* Interactive Reporting: Generates a Quarto-based dashboard with Plotly and Seaborn visualizations.



---

## Getting Started

### 1. Prerequisites

* Python 3.9+
* Quarto CLI
* Gemini API Key

### 2. Installation

Run the following command in your terminal:
pip install google-genai feedparser pytz reportlab pandas yfinance matplotlib seaborn plotly tabulate python-dotenv

### 3. Configuration

Create a file named .env in the root directory and add your key:
GEMINI_API_KEY=your_actual_api_key_here

### 4. Execution Pipeline

Step 1: Run the analysis script to generate data components:
python fin_news_dashboard.py

Step 2: Render the final HTML dashboard using Quarto:
quarto render market_analysis_dashboard.qmd

---

## Project Structure

* fin_news_dashboard.py: The Python engine for scraping and AI analysis.


* market_analysis_dashboard.qmd: The Quarto template for the visual interface.


* .env: Secure storage for API credentials.
* _top_drivers.md, _strategy_analysis.md, etc: Dynamic components generated by the script and included in the dashboard.



---

## GitHub Pages Deployment

1. Set output-dir: docs in a _quarto.yml file.
2. Render the project using the command: quarto render.
3. Push the docs/ folder to your repository.
4. Enable GitHub Pages in your repository settings, pointing to the /docs folder.
