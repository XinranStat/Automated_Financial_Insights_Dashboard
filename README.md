
# MarketSenseAI: Automated Financial Insights Dashboard

[cite_start]MarketSenseAI is an end-to-end data pipeline that scrapes global financial news, performs AI-driven sentiment analysis, and correlates findings with market data[cite: 1, 11]. [cite_start]It transforms raw reporting from Financial Times and The Economist into a strategic executive dashboard[cite: 1, 15].

---

## Core Features
* [cite_start]Automated ETL: Scrapes RSS feeds with custom time-filtering (48 hours for FT, 90 days for The Economist)[cite: 1, 15, 16].
* [cite_start]AI Classification: Tags articles into Macro, Tech/AI, Geopolitics, or Corporate Finance using Gemini 2.5 Flash[cite: 17, 18].
* [cite_start]Strategic Scoring: Assigns sentiment scores (-1.0 to +1.0) based on operational and strategic impact[cite: 1, 19].
* [cite_start]Market Correlation: Maps news events against Nasdaq 100 and 10Y Treasury yield volatility[cite: 1, 20].
* [cite_start]Interactive Reporting: Generates a Quarto-based dashboard with Plotly and Seaborn visualizations[cite: 11, 20, 21].



---

## Getting Started

### 1. Prerequisites
* [cite_start]Python 3.9+ [cite: 1]
* [cite_start]Quarto CLI [cite: 11]
* [cite_start]Gemini API Key [cite: 18]

### 2. Installation
```bash
# Install dependencies
pip install google-genai feedparser pytz reportlab pandas yfinance matplotlib seaborn plotly tabulate python-dotenv

```

### 3. Configuration

Create a .env file in the root directory:

```text
GEMINI_API_KEY=your_actual_api_key_here

```

### 4. Execution Pipeline

Run the analysis script to generate data components:

```bash
python fin_news_dashboard.py

```

Render the final HTML dashboard:

```bash
quarto render market_analysis_dashboard.qmd

```

---

## Project Structure

* fin_news_dashboard.py: The Python engine for scraping and AI analysis.


* market_analysis_dashboard.qmd: The Quarto template for the visual interface.


* .env: Secure storage for API credentials.


* _top_drivers.md, _strategy_analysis.md, etc: Dynamic components generated by the script and included in the dashboard.



---

## GitHub Pages Deployment

1. Set output-dir: docs in a _quarto.yml file.


2. Render the project using quarto render.


3. Push the docs/ folder to your repository.


4. Enable GitHub Pages in your repo settings, pointing to the /docs folder.
